Lab Challenges

First Contact - Hard choices (CTF)

Problem Statement:

The sky tore open as the ship descended, a gleaming monolith against the midday sun. Without warning, every screen flickered - phones, billboards, watches - replaced by a single message: "You are invited." A voice followed: "This is the first contact. Your species has been selected to participate in CyberGame, an interstellar CTF. Success grants you access to galactic science, post-scarcity tech, and diplomatic recognition. Failure means we proceed with scheduled construction of a hyperspace bypass that, regrettably, requires the demolition of your planet. Here is Message One."
Then a stream of cryptic data pulsed across the screens - elegant, alien, and waiting.
Additional instructions from the Earth Command:

  • Download the file.
  • Find a hidden message.
  • The messages are in the format SK-CERT{something} unless stated otherwise. Enter the full flag from the first "S" to the last "}".
  • Do not write anything else into the answer field; the aliens expect the exact flag.

Approach:

The flag was straightforwardly present in the `part1.txt` file provided. The primary step was to download and open the file, then search for the specified flag format.

Tools Used:

A standard text editor (to open `part1.txt`).

Flag(s) Found:

`SK-CERT{th1s-w0rks-w3ll}`

Screenshot:

Key Lessons Learned:

  • Always check the provided files thoroughly, even for seemingly simple challenges.
  • Pay close attention to specific flag formats and submission instructions.
  • Sometimes the simplest solution is the correct one.

SQL Injection Bypass Challenge

Problem Statement:

Bypass a login form that uses a vulnerable SQL query, without knowing the database structure, to gain administrative access.

Approach:

Used common SQL injection techniques like 'OR 1=1--' and UNION-based attacks. Identified potential database columns through trial and error, followed by data extraction.

Tools Used:

Burp Suite (for proxying and request manipulation), SQLMap (for automated discovery and exploitation once manual methods were understood).

Flag(s) Found:

`flag{YOUR_SQLI_FLAG_HERE}`

Screenshot:

Key Lessons Learned:

  • Understanding error-based and blind SQL injection types.
  • Importance of parameterized queries in preventing such attacks.
  • The power of systematic enumeration during exploitation.

Cross-Site Scripting (XSS) Reflected Challenge

Problem Statement:

Inject malicious JavaScript into a search functionality that reflects user input without proper sanitization, to steal cookies.

Approach:

Tested various HTML tags and JavaScript payloads within the search input. Used common XSS payloads like `<script>alert(document.cookie)</script>` and encoded variations.

Tools Used:

Web browser developer tools (console), Burp Suite (for URL encoding and request modification).

Flag(s) Found:

`flag{YOUR_XSS_FLAG_HERE}`

Screenshot:

Key Lessons Learned:

  • Importance of input validation and output encoding on the server-side.
  • Different types of XSS (reflected, stored, DOM-based).
  • How XSS can be leveraged for session hijacking and defacement.

Basic Buffer Overflow Exploitation

Problem Statement:

Exploit a vulnerable C program with a stack-based buffer overflow to execute arbitrary shellcode.

Approach:

Identified the vulnerable function and buffer size. Used GDB to determine the offset to the return address and crafted a malicious input with NOP sled, shellcode, and the overwritten return address.

Tools Used:

GDB (GNU Debugger), Python (for payload generation), objdump (for disassembling), Kali Linux environment.

Flag(s) Found:

`flag{YOUR_BOF_FLAG_HERE}`

Screenshot:

Key Lessons Learned:

  • Understanding stack frames and function call conventions.
  • The concept of return-oriented programming (ROP).
  • Mitigation techniques like ASLR, DEP, and canaries.
